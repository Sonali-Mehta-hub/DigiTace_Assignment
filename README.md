#  AI Powered Reply-Intent Classifier

Classify a creator's DM/email reply into an outreach intent category — powered by an LLM few-shot prompt via the **Groq API**, not a locally trained model.

## Overview

Talent and influencer agencies deal with a high volume of creator replies to outreach messages. Manually reading every reply to figure out whether a creator is interested, declining, asking about pricing, asking about availability, or just being vague is slow and doesn't scale.

This project classifies each creator reply into exactly one of five intent categories, using a single few-shot prompt sent to an LLM (via Groq's OpenAI-compatible API) — rather than a trained ML model. With a small hand-labeled dataset (60 examples), an LLM's few-shot reasoning generalizes better than a locally trained TF-IDF/Naive Bayes or logistic regression model would at this scale.

| Label | Description |
|---|---|
| `interested` | Creator is willing to collaborate |
| `not_interested` | Creator declines the collaboration |
| `pricing_query` | Creator asks about payment or budget |
| `availability_query` | Creator asks about campaign dates or scheduling |
| `unclear` | Reply doesn't clearly indicate intent |

Each prediction returns both the **predicted intent** and a **confidence score**.

## Features

- 🧠 Few-shot LLM classification via Groq (fast inference, generous free tier)
- 💬 Interactive **Streamlit** web UI to classify a single reply
- ⌨️ **CLI** for quick single-message or interactive-loop classification
- 📊 **Evaluation script** that runs the held-out test split and reports accuracy, per-category precision/recall, and a confusion matrix
- 🗂️ Small, hand-labeled dataset of realistic creator replies

## How It Works

```
Creator Reply
      ↓
Few-shot prompt (examples + system prompt)
      ↓
Groq LLM (chat completion, JSON response)
      ↓
Parsed { intent, confidence }
```

`classifier.py` builds a prompt containing a handful of labeled example replies plus the new message, sends it to Groq's chat completions endpoint, and parses the JSON response into a predicted intent and confidence score.

## Getting Started

### Prerequisites
- Python 3.10+
- A free [Groq API key](https://console.groq.com/keys)

### 1. Clone the repository
```bash
git clone https://github.com/Sonali-Mehta-hub/DigiTace_Assignment.git
cd DigiTace_Assignment
```

### 2. Create a virtual environment
```bash
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Set your API key
Create a `.env` file in the project root:
```
GROQ_API_KEY=your-key-here
```
Or export it directly in your shell:
```bash
export GROQ_API_KEY=your-key-here     # Windows: set GROQ_API_KEY=your-key-here
```

## Usage

### Streamlit app
```bash
streamlit run app.py
```
Opens at `http://localhost:8501`. Paste a reply in the **Classify a Message** tab to get a live prediction, or check the **Test Set Results** tab (requires running `evaluate.py` first).

### CLI
Single message:
```bash
python cli.py "What's your rate for a story post?"
```
Interactive loop:
```bash
python cli.py
```

### Evaluation
Runs the held-out test split (`data/dataset.csv`, rows marked `split == "test"`) through the classifier and reports accuracy, a confusion matrix, and per-category precision/recall. Also saves raw predictions to `results/test_predictions.json`.
```bash
python evaluate.py
```

## Dataset

- 60 hand-labeled examples of realistic creator replies (`data/dataset.csv`)
- Columns: `text`, `label`, `split`
- Split: 50 train / 10 test

The training rows aren't used to train a model — they're supplied to the LLM as few-shot examples inside the prompt.
